# SwiftKitchen AI — Landing Site

🚀 SwiftKitchen AI — The Smartest Way to Take Orders on WhatsApp
Turn your WhatsApp into a 24/7 automated order-taking machine. Built specifically for cloud kitchens, sweet shops, bakeries, and restaurants.


🔥 Why SwiftKitchen AI?
Restaurants lose up to 30% of sales due to delayed replies on WhatsApp. SwiftKitchen AI automates the entire process—from sending the menu to taking payments—without making your customers wait.


## 1. Tech Stack

This site is built with a modern full‑stack React setup, deployed on **Vercel** (edge runtime), without using Next.js.  

| Layer              | Choice                                                                 |
|--------------------|------------------------------------------------------------------------|
| Framework          | [TanStack Start v1](https://tanstack.com/start) (SSR + server functions) |
| UI                 | React 19 + TypeScript (strict)                                       |
| Routing            | TanStack Router (file‑based, type‑safe)                              |
| Bundler / server   | Vite 7                                                               |
| Language           | TypeScript (strict mode)                                             |
| Styling            | Tailwind CSS v4 (via `src/styles.css`)                               |
| Design tokens      | CSS custom properties in `oklch` color space                         |
| UI primitives      | shadcn/ui + Radix                                                    |
| Animation          | Framer Motion                                                        |
| Icons              | lucide‑react                                                         |
| Forms              | react‑hook‑form                                                      |
| Toasts             | sonner                                                               |
| Hosting target     | Vercel (Serverless / Edge Functions)                                 |

> Why not Next.js?  
> TanStack Start gives us first‑class type‑safe routing, server functions and edge‑native deployment without the App Router complexity.

## 3. Page Composition (Home `/`)

The landing page is a single scrollable route (`src/routes/index.tsx`) that renders in this order:

1. **Loader** — initial fade‑in.  
2. **Navbar** — fixed, glassmorphic, with logo + Book Demo CTA.  
3. **Hero** — headline, sub‑copy, two CTAs.  
4. **Marquee** — trusted‑by logos.  
5. **Problems** — 5 pain‑point cards.  
6. **WorkflowShowcase** — auto‑sliding workflow carousel.  
7. **Solution** — 4‑step automation flow.  
8. **LiveDemo** — full‑width auto‑sliding WhatsApp chat screenshots.  
9. **BuiltFor** — 6 capability cards.  
10. **Benefits** — 8 numbered benefit tiles on gradient.  
11. **Pricing** — Monthly/Yearly toggle, 3 tiers.  
12. **Testimonials** — auto‑sliding marquee, 6 quotes.  
13. **FAQ** — accordion.  
14. **ContactForm** — lead capture (name, email, phone, business, budget).  
15. **FinalCTA** — closing conversion section.  
16. **Footer** — logo, links, credit.  
17. **WhatsAppFab** — floating WhatsApp button.  
18. **ContactModal** — auto‑opens after 5s once per session.

## 4. Design System

All visual tokens live in `src/styles.css`. Use semantic Tailwind classes — never hard‑code colors in components.

- **Theme**: dark by default, deep indigo background `oklch(0.08 0.015 280)`.  
- **Primary**: `#ac6aff` — `oklch(0.68 0.22 305)` with a glow variant.  
- **Fonts**: `Playfair Display` (headings), `Inter` (body).  
- **Reusable utilities** (defined in `@layer components`):
  - `.glass` — translucent surface with blur + hairline border.  
  - `.glass‑card` — premium card with animated rainbow gradient ring.  
  - `.text‑gradient` — white→purple text gradient.  
  - `.btn‑primary‑glow` — primary gradient button with glow shadow.  
- **Gradients**: `--gradient‑primary`, `--gradient‑rainbow`, `--gradient‑hero`.  
- **Animations**: `marquee`, `pulse‑ring` keyframes for ambient motion.

Breakpoints follow Tailwind defaults (`sm` 640, `md` 768, `lg` 1024, `xl` 1280).

